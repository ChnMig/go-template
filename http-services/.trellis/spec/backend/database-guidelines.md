# Database Guidelines

> 本项目数据库相关约定。

---

## Overview

当前仓库没有数据库层实现，也没有引入 ORM、SQL builder、迁移工具或数据库驱动。代码检索未发现 `gorm`、`database/sql`、`sqlx`、迁移文件或 SQL 语句；`go.mod` 也未声明数据库相关直接依赖。

因此，未来任务不得假设已有数据库基础设施。需要接入数据库时，应先形成单独设计并补齐本文件，而不是在业务 handler 中临时创建连接或直接写 SQL。

---

## Query Patterns

当前无查询模式。未来新增数据库访问时遵循以下边界：

- API handler 不直接执行数据库查询。
- 领域层表达业务语义，持久化细节应放入独立 repository / storage 层或经明确设计后的等价结构。
- 查询函数应接收 `context.Context` 或可传递取消信号的上下文，不依赖全局请求对象。
- 事务边界应围绕业务用例设计，不能散落在多个 handler 中。

---

## Migrations

当前无迁移工具和迁移目录。新增数据库前必须先确定迁移方案，并记录：

- 迁移文件目录。
- 命名规则。
- 本地执行命令。
- CI / 发布流程中的执行方式。
- 回滚策略。

---

## Naming Conventions

当前无表、列、索引命名约定。接入数据库时，应在首个数据库任务中通过真实实现补充，而不是提前写入未验证的理想规则。

---

## Common Mistakes

- 不要因为需要持久化就直接在 `api/app/...` handler 中打开数据库连接。
- 不要在 `utils/` 下放业务特定的数据访问逻辑；`utils/` 只放跨业务通用能力。
- 不要引入 ORM 或迁移框架后只更新代码、不更新本规范和 `.agentdocs/index.md` 中的长期记忆。
